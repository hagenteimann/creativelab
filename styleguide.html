00 Brand Vision

Premium Content. Ohne Overhead.

creativelab.de steht für High-End Videografie und Content-Strategie im Fashion- und Lifestyle-Segment. Unsere visuelle Sprache ist minimalistisch, präzise und wertig. Dieses Manual definiert die Regeln, um diese Ästhetik über alle Touchpoints hinweg konsistent zu halten.

01 Logo & Branding

Das Kern-Logo

Unser Logo ist das zentrale Identitätsmerkmal. Es sollte immer mit ausreichend Weißraum (Clear Space) platziert werden.

Negativ (Standard) — Full Background
Positiv — Light Background
Minimale Breite24px (Digital) / 10mm (Print)
Clear SpaceMindestens 50% der Logo-Breite rundherum
FarbvarianteAusschließlich Weiß oder Schwarz (Invertiert)

02 Foundations

Grid System — 4pt Tokens, 8pt Layout Base

Entscheidung: 4pt-Tokens sind ein Superset von 8pt (bewährt bei Linear, Vercel, Radix). Layout-Abstände nutzen immer 8pt-Vielfache (--space-2, --space-4, --space-8…). --space-1 und --space-3 sind ausschließlich für Micro-Details (Iconpadding, Borders, enge Typografie).

--space-1 · Micro only4px
--space-2 · Layout ✓8px
--space-3 · Micro only12px
--space-4 · Layout ✓16px
--space-6 · Layout ✓24px
--space-8 · Layout ✓32px
--space-12 · Layout ✓48px
--space-16 · Layout ✓64px

Fluid Spacing (clamp)

Section-Level-Abstände skalieren stufenlos ohne Media Queries. Niemals feste px-Werte für Section-Padding verwenden.

--sp-sectionclamp(80px, 10vw, 140px) — Section padding top/bottom
--sp-innerclamp(24px, 5vw, 56px) — Inner component padding
--sp-pageclamp(24px, 6vw, 100px) — Horizontal page gutter

04 Light Mode Accessibility

Mixed Mode Strategy

Um eine "Premium Cinematic" Ästhetik zu wahren, bleiben Schlüssel-Sektionen (Hero, Services, Pricing, Footer) auch im Light Mode **vollständig dunkel**.

Default ThemeLight Mode (Positiv)
Dark SectionsHero, Services, Pricing, Footer
Light SectionsAbout, Portfolio, Use-Cases, Contact

Readability Rules

Um die Lesbarkeit im Light Mode zu gewährleisten, müssen adaptive Tokens verwendet werden, außer in "Dark Sections".

InversionNav & Buttons invertieren nur außerhalb von Dark Sections
InputsLight Background (#f9f9f9) + Border (#eeeeee)
ContrastMindestens 4.5:1 (WCAG AA)

07 Motion & Interaction

Duration Tiers

Entscheidung: Blueprint (200–300ms für Micro) hat Recht. Alte Angabe (300–600ms) war zu langsam für Feedback-Loops. Drei Tiers per Material Design + Apple HIG.

--duration-micro · 150msHover, Click-Feedback, Icon-Swap
--duration-component · 250msDropdowns, Tooltips, State-Changes, Button-Press
--duration-page · 450msModals, Drawer, Page-Transitions, Hero-Reveal

Easing Curves

--ease-standardcubic-bezier(0.4, 0, 0.2, 1) — Standard für Hover & State
--ease-entercubic-bezier(0.0, 0, 0.2, 1) — Decelerate: Elemente erscheinen
--ease-exitcubic-bezier(0.4, 0, 1, 1) — Accelerate: Elemente verschwinden

Skeleton Screens

Für alle asynchron geladenen Inhalte (Bilder, Listings, Daten) Skeleton Screens statt Spinner verwenden. Skeleton verhindert Layout-Shift und signalisiert Struktur.

AnimationShimmer-Gradient, 1.5s infinite, ease-in-out
FarbeHintergrund: neutral-800 → neutral-700 (Shimmer)
RadiusGleicher Radius wie finales Element

10 Assets & Performance

Bildformate

Konsequente Nutzung moderner Formate — kein JPG/PNG ohne Konvertierung.

Fotos / RendersWebP. Auto-Convert bei Implementierung. Fallback: JPG.
Icons / LogosSVG. Inline wenn animiert, external wenn statisch.
Aspect RatioImmer feste Ratios (object-fit: cover). Niemals frei skalieren — verhindert CLS.
RatiosBento: 1:1, 16:9, 4:5 — je nach Slot-Typ

Scrim-Regel

Text-Overlays auf Bildern immer mit sanftem Verlauf absichern. Niemals Text direkt auf Bild ohne Scrim.

Gradientlinear-gradient(to top, rgba(0,0,0,0.40) 0%, transparent 60%)
Max Opacity40% Black — darüber wird der Scrim dominant
Kontrast-CheckWCAG 4.5:1 auch mit Scrim verifizieren

11 Touch & UX

Touch Targets

Mobile-First. Alle klickbaren Elemente müssen die Mindestgröße einhalten — auch wenn das Element visuell kleiner ist (Padding als Buffer nutzen).

--touch-min48 × 48px — WCAG 2.5.5 + Material Design Standard
Visuell kleinmin-height: var(--touch-min) + padding ausgleichen
Gap zwischen TargetsMindestens 8px (--space-2)

Safe Areas (Mobile)

CTAs und interaktive Elemente niemals in der Notch-Zone oder unter der Home-Bar platzieren.

Bottom CTApadding-bottom: max(var(--space-6), env(safe-area-inset-bottom))
Top Navpadding-top: max(var(--space-4), env(safe-area-inset-top))
Viewport-Höhedvh (dynamic viewport height) statt vh für mobile Browser
